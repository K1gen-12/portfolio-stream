# Personal Web Site (Blog + Lab)

このリポジトリは **Codex（AI）を使ってコードを追記・修正しながら育てていく個人Webサイト** です。  
完成形を一気に作るのではなく、**小さな変更を積み重ねて育てる**ことを前提にしています。

ブログ機能を持ちつつ、  
- 思考ログ  
- 個人開発の途中経過  
- 試行錯誤のメモ  

をそのまま残すための「個人ラボ」的なサイトです。

---

## コンセプト

> 作りながら考えたことを、そのまま置いていく場所

- 未完成でも公開する
- 途中の迷い・変更理由も残す
- 後から見返して「自分のOS」になることを重視

## Codex（AI）利用方針

このリポジトリでは、AIにすべてを任せず「補助」として使う運用を重視します。

- 既存コードの構造と命名を尊重する
- 1プロンプト = 1変更（変更は小さく・差分は最小限）
- 変更点は「どのファイルをどう変更したか」を明示する

以下はよくある要望と要件の例です。実装依頼時は目的と要件を明確にしてください。

1) 新しい静的ページを追加
 - 要件: App Router構成に従い既存レイアウトを流用。パス: `/about`、簡単な説明を表示。

2) ブログ記事一覧の改善
 - 要件: `content/blog` のMarkdownを使用し、日付の新しい順に表示。タイトル／日付／タグを表示。

3) 記事詳細ページの微調整
 - 要件: タイトル下に日付とタグを表示。Markdown本文はそのまま活かす。

4) frontmatter の拡張
 - 追加項目例: `description`（要約）、`draft`（true の場合は一覧に表示しない）

5) lib 層（ロジック）の整理
 - 要件: `lib/blog.ts` に型定義を追加し、frontmatterの型安全性を向上させる（挙動は変えない）。

6) デザイン調整（安全版）
 - 要件: Tailwindのユーティリティのみ使用して色・余白を調整。新しいコンポーネントは追加しない。

7) 大きな変更の前提
 - 注意: 影響範囲が広い変更は、実装前に方針を箇条書きで提示してください。

8) 変更のリカバリ
 - 今の変更を元に戻す手順を用意してください（コミット単位での復元を想定）。
